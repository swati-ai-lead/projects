# Step-by-step plan for EB1 GitHub contribution journey

## Phase 1: Foundation
1. Create a GitHub account or use swati-ai-lead.
2. Create a main repository for your portfolio projects.
3. Organize projects into folders: 01-beginner-projects, 02-intermediate-projects, 03-advanced-projects.
4. Keep each project self-contained with its own README.

## Phase 2: First beginner project
1. Choose a simple project such as a to-do list or personal portfolio.
2. Build it using one technology you are comfortable with.
3. Add clear documentation and screenshots.
4. Commit in small steps and push to GitHub.

## Phase 3: Build evidence for EB1
1. Record the problem solved, the feature built, and the impact.
2. Keep a log of dates, commits, and project descriptions.
3. Make sure each project shows originality, usefulness, and documentation.
4. Use the repository as evidence of sustained technical contribution.

## Phase 4: Scale gradually
1. After 2-3 small projects, move to slightly more complex ones.
2. Add automation, APIs, tests, or deployment.
3. Continue improving documentation and presentation.
4. Maintain consistency rather than rushing.
